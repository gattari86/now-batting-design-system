02 / Color System

Sunlight first, neon only for moments.

The palette is a warm off-white base, a disciplined text ramp, and two accents doing two very different jobs. Neon is emotional. Daylight is functional. Never confuse them.

Rule 01
Never put neon on pure white for text.
The contrast ratio fails outdoors and fails accessibility. For any text that must be read, put neon on dark. Always.
Rule 02
Daylight accent for buttons. Neon for moments.
Daylight (B8CC00) holds up in direct sun and reads as a CTA. Neon (DFFF00) is reserved for the one thing happening on screen right now.
Rule 03
State is never communicated by color alone.
Active rows get a neon bar, tinted background, and bolder weight. If a user is colorblind or in glare, the hierarchy still works.

Hierarchy rules
Every text decision answers one question.

Player names bold. Song titles medium. Metadata muted gray. The hierarchy is the UI.

Minimum body 16px. Do not shrink type to fit data. If it does not fit, cut the data.

H1 is for moments, not every screen. One hero statement per flow. Everywhere else, step down.

JetBrains Mono is a garnish, not a voice. Use it for labels, specs, and numbered steps. Never for body copy.

Six rules for every component.

  • 44px minimum tap target. Gloves-on friendly.
  • 12-16px radius. Soft but not bubbly.
  • 1px borders, not shadows. Outdoor glare breaks drop shadows.
  • One primary per screen. Never two fighting CTAs.
  • Dark play buttons. Neon on dark carries the sound-is-alive feeling.
  • Icons match in stroke and corner. Mixing styles looks amateur.
